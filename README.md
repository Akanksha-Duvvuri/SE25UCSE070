## Setup Instructions

1. Clone the repo and `cd` into the folder.
2. Install dependencies:
```bash
   npm install
```
3. Make sure MongoDB is running locally on `127.0.0.1:27017`.
4. Start the server:
```bash
   node index.js
```
5. Visit `http://localhost:8080`.

## Database Schema

| Field | Type | Notes |
|---|---|---|
| `_id` | ObjectId | Auto-generated by Mongoose|
| `originalFileName` | String | The filename as uploaded by the user |
| `storedFileName` | String | `_id` + original extension — the actual name on disk |
| `fileSize` | Number | Size in bytes |
| `MIMEType` | String | From Multer's detected mimetype |
| `uploadDate` | Date | Defaults to upload time / current time in IST |

## Architecture Overview

```
1. Browser 
2. Express routes (index.js) 
3. Multer (parses multipart upload) 
4. Writes file to "uploads/" 
5. Saves metadata row to MongoDB

```

File bytes live on disk in `uploads/` and the metadata lives in MongoDB, in the `formats` collection.

## Design Decisions

- **Filename collisions:** The mongodb auto generated id is used as the file's actual name on the disk to prevent file name collisions.
- **Write ordering:** The file is written to disk *before* the DB row is inserted. If the DB insert then fails, the code deletes the just-written file (`fs.unlinkSync`) so no file is left. If the disk write itself fails, no DB row is ever created.
- **Delete ordering:** The DB row is deleted first, then the file on disk. If the disk delete fails afterward, there will be a harmless leftover file and no DB record pointing to that file
